# 方向 3：公告冲击量级评估 — PLAN

> 当上市公司发布重要公告时，自动评估该公告对雪球讨论区情绪的量化影响。
> 不是"公告说了什么"，而是"公告发布后市场真实反应是什么"。

---

## 1. 背景与动机

### 1.1 一个真实案例

```
2026-05-28 茅台发布 2025 年度利润分配方案（8017 字 PDF）
  ├─ 每 10 股派发现金 238 元
  └─ 讨论题从"批价跌破2500、内需萎靡"转为"分红率 vs 估值"

问题：如果没有跑完整分析，怎么知道这次公告到底有没有改变市场看法？
```

### 1.2 解决什么问题

| 场景 | 现在的做法 | 方向 3 的做法 |
|------|-----------|-------------|
| 我的持仓股发了季报 | 手动打开雪球刷帖子，凭感觉判断情绪 | 系统告诉我：Q vs Q 讨论量 +X%, 情感 +Y, 热词从"内卷"→"分红" |
| 自选股有回购公告 | 通知栏看到"XXX 发布回购公告"，忽略 | 系统分析后推送：8 条回购公告累计效果 → 负面讨论 -15% |
| 想对比两只股票的"财报冲击力" | 没有对比工具 | 并列对比：茅台利润分配 → 情绪提升 0.2 vs 泡泡玛特 Q1 数据 → 情绪提升 0.4 |

### 1.3 与方向 2 的关系

| | 方向 2 情绪看板 | 方向 3 公告冲击 |
|--|---------------|----------------|
| 触发 | 每天定时 | **公告事件驱动** |
| 视角 | 64 只全景 | **单股焦点事件** |
| 核心问题 | "今天市场在关心什么" | "这个公告到底有没有用" |
| 补充关系 | 日报告诉你茅台今天异动 | 冲击分析告诉你异动是不是公告带来的 |

## 3. 技术方案

### 3.1 核心逻辑

```
公告事件（如"茅台利润分配方案"）
    │
    ▼
时间锚点：
  ├─ T = 公告发布时间
  ├─ T-24h = 公告前 24h
  └─ T+24h = 公告后 24h
    │
    ▼
查询 crawl_snapshots：
  ├─ T-24h ~ T：基线讨论量 + 情感均值
  ├─ T ~ T+24h：冲击后讨论量 + 情感均值
  └─ 变化量 = (后 - 前) / 前 × 100%
    │
    ▼
输出：
  ├─ 讨论量变化: +65%
  ├─ 情感偏移: -0.2 (偏空)
  ├─ 热词迁移: "回购"→48% → "内卷"→32%
  └─ 冲击等级: 低/中/高 (基于 Z 分数)
```

### 3.2 数据结构

```python
@dataclass
class AnnouncementImpact:
    symbol: str
    announcement_title: str
    announcement_time: int           # Unix timestamp
    announcement_type: str           # 季报/回购/分红/管理层/其他
    content_length: int              # PDF 字数

    # 公告前基线
    baseline_period: str             # "2026-05-27 00:00 ~ 2026-05-28 00:00"
    baseline_post_count: int
    baseline_sentiment_avg: float
    baseline_hotwords: List[str]

    # 公告后冲击
    impact_period: str               # "2026-05-28 00:00 ~ 2026-05-29 00:00"
    impact_post_count: int
    impact_sentiment_avg: float
    impact_hotwords: List[str]

    # 变化量
    post_delta_pct: float            # 讨论量变化 %
    sentiment_delta: float           # 情感偏移
    hotword_shift: List[tuple]       # ("回购", +0.15) 热词频次变化
    impact_level: str                # high(Z>3) / medium(Z>2) / low
```

### 3.3 公告类型检测

基于现有 `_detect_notice_type()` 的规则，结合 PDF 标题提取：

| 类型 | 检测规则 | 分析重点 |
|------|---------|---------|
| 季报/年报 | title 含"季度报告""审计报告" | 营收/利润/利润率变化 → 讨论转向 |
| 回购 | title 含"股份购回""回购" | 连续回购的累计情绪效果 |
| 分红 | title 含"利润分配""股息" | 分红率 vs 估值讨论 |
| 管理层变动 | title 含"董事会""独立董事""聘任" | 市场对管理层信任度变化 |
| SEC 备案 | title 含"Form 4""6-K""20-F" | 内幕交易信号（增减持方向） |

### 3.4 数据来源

```
公告数据:
  ─ xueqiu-analyzer data/*.json 的 notices[] 字段 → 公告列表
  ─ xueqiu-monitor announcements 表 → 补充监控期的公告

讨论数据:
  ─ xueqiu-monitor crawl_snapshots 的 posts_data → 帖子级时间戳 + 情感分

运行时:
  ─ 脚本运行时读一次 DB，按时间窗口切片
  ─ 无须实时流处理
```

### 3.5 输出格式

```markdown
📊 公告冲击报告 — 茅台 2025年度利润分配方案
═══════════════════════════════════════

 公告信息
 ────────
 股票: 茅台 SH600519
 公告: 2025 年度利润分配方案（10 派 238 元）
 发布: 2026-05-28 16:40
 正文: 8017 字
 类型: 分红

 冲击评估
 ────────
 讨论量: 公告前 48 条/天 → 公告后 79 条/天  (+65% 🔺)
 情感值: 公告前 -0.05 (中性偏空) → +0.12 (中性偏多)
 冲击评级: 中等 (Z=2.1)

 关键变化
 ────────
 ✅ 热词迁移
   "批价" 前 32% → 后 18%  ↓ 不再关注
   "分红" 前 12% → 后 45%  ↑ 新焦点
   "回购" 前 28% → 后 35%  ↑ 持续关注
   "内卷" 前 25% → 后 15%  ↓ 讨论减退

 📌 结论
 本次分红公告成功将讨论焦点从"批价下行/内卷"转向"分红回报"。
 但情感值仅从 -0.05 升到 +0.12，说明市场仍在观望，未完全转为乐观。
 建议关注下周是否有进一步的回购公告配合。
```

## 4. 执行方案

### 方案 A：离线历史分析（先验证可行性，5 天）

不依赖实时事件流，基于已有数据先做一次完整的公告冲击回溯分析。

**输入**：当前 `data/*.json` 已有的茅台/泡泡玛特/PDD 公告

| 公告 | 预期发现 |
|------|---------|
| 茅台利润分配方案（10 派 238 元） | 讨论从"批价"转向"分红" |
| 泡泡玛特 8 条回购 | 连续回购 → 正面情绪积累 |
| PDD Q1 营收 1062 亿 | 财报后讨论从"仅退款"转向"利润率" |

**输出**：三份历史公告冲击报告 + 一份方法论总结

**目的**：验证分析框架是否有效，再决定是否做成实时功能。

### 方案 B：在线监控分析（需要实时事件流，2 周+）

**输入**：xueqiu-monitor 每日快照 + xueqiu-analyzer 的公告爬取

**流程**：
1. 每天检查 xueqiu-monitor 的 announcements 表，发现新公告 → 标记时间锚点
2. 累计 48h 数据后 → 生成公告冲击报告
3. 推送到飞书

**推荐路径：先做方案 A 验证，再决定是否需要方案 B。**

## 5. 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 公告时间锚点不精确（只精确到日） | 高 | 中 | 使用公告 title 中的具体时间（如"16:40"） |
| 同时发生多个公告（如季报+分红同日） | 中 | 中 | 无法区分各自贡献，合并分析 |
| 讨论区 24h 内有外部事件干扰（如大盘暴跌） | 中 | 高 | 需要大盘对照基线（对比同板块其他股票） |
| 美股 SEC 公告无 PDF 内容 | 高 | 低 | 至少标题和类型可用 |
