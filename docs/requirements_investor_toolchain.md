# 价值投资者雪球工具体系 — 需求文档

> 种子来源: `docs/seed_investor_toolchain.md`
> 最后更新: 2026-06-02

---

## 1. 一句话描述

将本地 5 个独立项目（舆情监控 + 深度分析 + 财务估值 + 专栏追踪 + AI 助手）整合为统一的价值投资决策体系，通过飞书 Bot 统一交互，实现「发现信号 → 深度研究 → 财务估值 → 决策输出」的闭环。

---

## 2. 目标用户

| 角色 | 定义 | 核心需求 |
|------|------|----------|
| 个人价值投资者 | 管理 50-100 只关注股，低频交易，深度研究驱动 | 自动化研究流程，减少手动操作，加快决策速度 |
| 投资研究爱好者 | 追踪特定分析师/KOL 观点，积累投资知识 | 知识沉淀 + 交叉验证 |

---

## 3. 核心价值

**现状**: 工具各自独立运行，用户需要手动在多个项目中切换。

**目标**: 飞书成为唯一交互入口。用户在飞书群看到舆情告警 → 点击按钮触发深度分析 → 几分钟后收到 AI 报告 + 财务估值 → 直接在飞书上做决策。

---

## 4. 功能需求

### 4.1 持仓雷达（每日自动）

**用户故事**: 作为价值投资者，我每天早上打开飞书，就能看到持仓股票的情绪摘要 + 异常告警，点击异常股票可以一键生成综合研究报告。

**触发**: cron 每日运行
**流程**: xueqiu-monitor → 飞书推送摘要 → 用户点击深度分析 → xueqiu-analyzer + financial-sdk → 飞书推送报告

| 阶段 | 工具 | 输出 |
|------|------|------|
| 监控 | xueqiu-monitor | 7:00 cron 跑 64 只股票，P0 告警推送飞书 |
| 日报 | xueqiu-monitor | 爬取摘要 + 异常汇总 |
| 深度分析 | xueqiu-analyzer-skill | AI 投资报告 (8 维度) |
| 估值 | financial-sdk | F-Score / DCF / Graham Number |

**验收标准**:
- [ ] 飞书每天早上收到舆情摘要
- [ ] P0 告警包含「深度分析」飞书卡片按钮
- [ ] 点击按钮后 5 分钟内收到 AI 报告 + 财务估值
- [ ] 无告警时只发日报摘要，不推送

---

### 4.2 事件驱动（公告影响评估）

**用户故事**: 持仓公司发布公告时，自动通知并评估影响。

**触发**: xueqiu-monitor 检测到 new_announcement
**流程**: 公告检测 → financial-sdk 获取财务快照 → LLM 综合评估 → 飞书推送

| 输入 | 处理 | 输出 |
|------|------|------|
| 公告标题 + 公告类型 | financial-sdk 获取 ROE/PE/负债率/股息率 | 评估: "回购 10 亿 (市值 1.5%)，ROE 高，社论正面" |
| 公告内容 | LLM 提取公告要点 | 3 句话摘要 |

**验收标准**:
- [ ] 公告检测后自动获取财务快照
- [ ] LLM 评估影响并附上理由
- [ ] 飞书消息包含「正/负/中性」标记 + 摘要

---

### 4.3 选股筛选（手动触发）

**用户故事**: 我在飞书上输入行业/关键词/估值范围，系统返回符合条件的股票池。

**触发**: 用户通过飞书消息发送指令
**流程**: 筛选条件 → xueqiu-monitor 获取关注度 → financial-sdk 估值筛选 → 飞书推送候选池

| 筛选条件 | 工具 | 实现方式 |
|---------|------|---------|
| 行业/赛道 | xueqiu-monitor | 热词检测 + 行业标签 |
| F-Score >= 7 | financial-sdk | Piotroski F-Score 计算 |
| PE < 20 | financial-sdk | 最新 PE 查询 |
| 舆情偏正面 | xueqiu-monitor | sentiment_mean > 0.1 |

**验收标准**:
- [ ] 飞书发送 "筛选 消费行业 F-Score>=6" → 返回候选池
- [ ] 每只股票显示基本估值指标 + 舆情热度
- [ ] 结果可排序、可选择

---

### 4.4 知识沉淀（专栏追踪）

**用户故事**: 追踪特定雪球作者的专栏文章，自动生成摘要，与持仓股票交叉匹配。

**触发**: cron 每日运行
**流程**: xueqiu-crawler 爬取 → LLM 摘要 → 匹配持仓股票 → 飞书推送「该文章与你持仓的 XX 股票相关」

**验收标准**:
- [ ] 每天自动爬取关注的雪球作者文章
- [ ] AI 生成 3 句话摘要
- [ ] 自动检测文章中是否提到持仓股票
- [ ] 如果提到 → 飞书推送，附上「深度查一下」按钮

---

### 4.5 飞书 Bot 统一入口

**用户故事**: 所有工具通过一个飞书 Bot 交互，不需要打开任何终端。

**指令示例**:
| 飞书消息 | 效果 |
|---------|------|
| `分析 600519.SH` | xueqiu-analyzer + financial-sdk → 综合报告 |
| `估值 0700.HK` | financial-sdk → F-Score + DCF + Graham |
| `筛选 F-Score>=7 PE<15` | 选股筛选 → 候选池 |
| `舆情 贵州茅台` | xueqiu-monitor → 近 7 日舆情趋势 + 情感分布 |
| `日报` | 今日舆情摘要 |

**验收标准**:
- [ ] Bot 响应用户发送的消息
- [ ] 支持自然语言输入（如 "看看茅台的估值"）
- [ ] 响应时间 < 2 分钟

## 5. 技术架构

```
                        ┌──────────────────┐
                        │   飞书 Bot 入口    │
                        │  (lark event 订阅) │
                        └───────┬──────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                  ▼
     ┌────────────┐    ┌──────────────┐    ┌──────────────┐
     │ 调度中心    │    │ 知识库       │    │ 通知渠道    │
     │(事件路由)   │    │(飞书文档)    │    │(lark CLI)   │
     └─────┬──────┘    └──────────────┘    └──────────────┘
           │
     ┌─────┼──────┬──────────┬──────────────┐
     ▼     ▼      ▼          ▼              ▼
  monitor analyzer crawler financial-sdk langchain
 (舆情)  (分析)  (专栏)    (估值)       (Agent)
```

### 关键集成点

| 集成 | 方式 | 说明 |
|------|------|------|
| monitor → analyzer | CLI 调用 | `cd ../xueqiu-analyzer && python main.py 600519.SH` |
| monitor → financial-sdk | Python import | `from financial_sdk import ...` |
| Bot → 所有工具 | lark event + 飞书卡片 | 消息回调 + 按钮交互 |
| 输出 → 飞书 | lark CLI | `lark-cli im +messages-send` |

**工期**: 1-2 周
